# Audio Upload Web Design

**目标**

构建一个公开可访问的极简网页，允许用户从手机浏览器上传本地音频文件到云端；后续树莓派或电脑可从同一云端目录拉取文件并播放。

**推荐架构**

- 前端：Cloudflare Pages 托管静态上传页
- 上传接口：Cloudflare Pages Functions
- 文件存储：Cloudflare R2
- 后续播放端：树莓派或电脑轮询 R2 的 `incoming/` 目录

**为什么选这套**

- 用户只需要打开一个网址，不依赖 App
- 不需要自建服务器
- 2 GB 级别容量需求在免费额度内可行
- 后续把电脑轮询切换为树莓派轮询时，不需要重做网页

**用户流程**

1. 用户打开公开网页
2. 选择一个本地音频文件
3. 点击上传
4. 页面显示上传中进度与结果
5. 上传成功后，文件进入 R2 的 `incoming/` 目录

**功能边界**

当前阶段只实现：

- 上传网页
- 音频类型与大小校验
- 成功/失败提示
- 上传到云端对象存储

当前阶段不实现：

- 用户登录
- 上传历史查询
- 树莓派自动播放
- 播放队列管理

**文件规则**

- 接受格式：`mp3`、`m4a`、`wav`
- 文件大小：默认限制为 `50 MB`
- 对象命名：`incoming/<timestamp>-<safe-filename>`

**异常处理**

- 非音频文件：前后端双重拒绝
- 文件过大：前后端双重拒绝
- 文件名冲突：后端追加时间戳，避免覆盖
- 上传失败：页面显示明确错误，不返回伪成功

**部署方式**

- 开发期：本地静态预览 + 本地测试
- 上线期：部署到 Cloudflare Pages，使用默认 `*.pages.dev` 域名
- 存储配置：在 Cloudflare 控制台创建 R2 Bucket，并绑定到 Pages Functions
